  9. CSS - defines the look of your web site (html defines the content). One way to see this is to look at any web site (using firefox, other browsers have the option as well) and choose View, Page Style, No Style. This will remove all the style features and show you just the plain web page. A good site to see this is www.csszengarden.com. Click on the various links that will change the CSS style and show you the same content only presented in many VERY different ways.
    1. Create style rules to define the look. the format is always: 
      selector { property: value; [property: value];}
      for example:
      body { background-color: #ffffff; }
    2. CSS: background-color: hex code for color - used to change the background color for an element.
      Ex. 
      body { background-color: #ff0000; }
      will produce a red background color.
      Colors:
      red = #ff0000
      green = #00ff00
      blue = #0000ff
      white = #ffffff
      black = #000000
      magenta = #ff00ff
      yellow = #ffff00
      Browser safe colors
    3. background-image: url(redbg.gif); is used to load an image for your background where redbg.jpg is the name of the image file. Image files are generally very small and repeat themselves across the screen. It is also common to have a background-color that closely matches the color of your image. The reason for this is text loads first and the background image like all images load last. This enables people to read your text a little better.
    4. Inline Style - located on the line that you want the style to occur. As a RULE we don't use inline style but instead would use an ID (explained soon). Notice that this list item is IV instead of four because I used an in-line style for this list item:
      <li style="list-style-type: upper-roman; font-weight: bold;
  padding-top: 1em;">

      New way would be:
      CSS 
      #specialList {
   list-style-type: upper-roman;
   font-weight: bold;
   padding-top: 1em;
}
      html 
      <li id="specialList">
    5. Embedded Style (document style) - located in the HEAD section of the document will change the style for the whole document. As a RULE we don't use an embedded style we give the body tag an ID attribute and use what is called a Descendant Selector 
      <style type="text/css">
ol li{ list-style-type: upper-roman; font-weight: bold; 
  padding-top: 1em;}
ol ol li{ list-style-type: decimal; font-weight: normal;}
</style>

      New way would be:
      CSS 
      #home ol li{
   list-style-type: upper-roman;
   font-weight: bold;
   padding-top: 1em;
}
      html 
      <body id="home">
    6. Linked Style Sheet - that can be linked to all your documents (best way to format a web site) !-- HTML generated using hilite.me -->
      <link rel="stylesheet"
href="notestyle.css"
type="text/css"
media="screen" />

      The file is called notestyle.css and only has the style and no html tags at all.
    7. Precedence is in-line, document, linked sheet.
    8. id - an id is used to uniquely identify one element on a page. It is common to give the body element and id reflecting the name of the page, like home, garden etc.

      Ex: 
      <body id="home">


      you create a style rule using a #nameofID ex:

      #home{
	background-color: #ff0000;
}
      this will change the background color for just the element with an id of home. You can use an id name only once per page but you can use it on more than one page. In the case of home for the body tag it only makes sense to use it on the home page.
    9. Descendant Selector - is a way to style an element that is a descendent of another element. an example is the easy way to explain it. If you want to style the h1 tag on only the home page (which we gave an id in the body tag equal to home) you would write this rule:
      #home h1{
	border-bottom: thick solid black;
}
      what happens is only the h1 tags that are in the "box" that has an id of home will be effected.
    10. class is when you want a set of CSS rules to be used more than once per page for example: 
      .first { text-indent: 3em; }

      Creates a class named first that indents the first line 3 capital letter M's.
      This line is indented. Notice how the effect does not carry over to the second line. This is why I named it 'first' for first line indent! You use this class by using the class attribute of the tag: 
      <p class="first">This line is indented. ...
      as many times on a page as needed.
